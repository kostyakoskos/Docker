Данные в докер могут хранится временно или постоянно:
Временно. По умолчанию, файлы созданные приложением, работающие в контейнере, сохраняются в слое контейнера поддерживающего запись. После того как контейнер перстанет 
существовать, данные исчезнут.
tmpfs- можно подключить к контейнеру для повышения производительности. При этом данные, будут хранится временно.
Постоянно. Данные можно хранить и после уничтожения контейнера. Есть 2 способа хранить данные постоянно: 1) Примонтировать к контейнеру реально существующую папку. Так делать 
плохо, т.к. 2 есть сложности когда 2 и более контейнера используют одну и туже папку. Миграция данных также проблема. Поэтому лучше использовать Тома. 2) Том - файловая 
система на хост машине, расположенная за пределами контейнеров.
Тома: 
Они представляют собой средства для постоянного хранения информации.
Они самостоятельны и отделены от контейнеров.
Ими могут совместно пользоваться разные контейнеры.
Они позволяют организовать эффективное чтение и запись данных.
Тома можно размещать на ресурсах удалённого облачного провайдера.
Их можно шифровать.
Им можно давать имена.
Контейнер может организовать заблаговременное наполнение тома данными.
Они удобны для тестирования.
Создавать тома в docker можно использую формат JSON
Сетевая подсистема Docker — среда, которая позволяет организовывать взаимодействие контейнеров.
Docker Compose — технология, упрощающая работу с многоконтейнерными приложениями.
Docker Swarm — средство для управления развёртыванием контейнеров.
Сервисы Docker — контейнеры в продакшне.
Конетейнеры состоят из слоев. Каждый слой кроме последнео предназначен только для чтения. Слой это файл который описывает изменения после последнего слоя.
При скачивании какого-либо репозитория, докер скачивает только слои которых нет.

docker run vs dovker create. Разница в том, что create создает контейнер в неработающем состоянии. А run создает и запускает контейнер
Какая разница между виртуализацие и контейнеризацией? Разница в том, что при виртуализции на одном сервере можно запустить несколько операционных систем. А при контейнеризации на одной
операционной системе 

!Сначала делаем образ, а потом запускаем его в контейнере(run=create+start)!

Каждый слой это закешированная часть файловой системы. Надо создавать минимум слоев Docker. Например, можно писать все LABEL в 1 строку. Если будет 2 Label в 2 строки, docker 
создаст 2 слоя, что плохо.
Докер файл может создавать несколько вариантов запуска. Например для тестирования и для запуска. Как он будет запускаться передаем в качестве аргументов.
